System Overview

The Salesforce DocGen Platform is a highly scalable, fully native document generation engine. It provides UI components for end-users, background asynchronous pipelines for bulk processing, and a zero-config PDF rendering engine.

Template Syntax & Logic

The document engine parses raw OpenXML inside .docx and .pptx files. It supports simple merges, related list loops, and conditional visibility.

1. Simple Merge Fields

Insert a standard field value by wrapping the API name in braces. Supports cross-object spanning defined in your query configuration.

Hello {FirstName} {LastName},
Your account {Account.Name} has a balance of {Amount}.

2. Conditional Visibility (Boolean Logic)

You can hide or show entire sections of a document based on a Boolean (Checkbox) field or a Formula field that evaluates to true/false.

Pro Tip: Use Salesforce Formula Fields (Checkbox return type) to evaluate complex business logic (e.g., Amount > 10000 && StageName = 'Closed Won') and use that formula field in your template for conditional visibility!

To use conditional visibility, wrap the content in hash blocks using the boolean field's API name:

{#Is_High_Value_Customer__c}
    Thank you for being a premium VIP member! You receive a 20% discount.
{/Is_High_Value_Customer__c}

If Is_High_Value_Customer__c is false, the entire block (including the tags) is removed from the final document. If true, the tags are removed and the content remains.

3. "Smart" Table Row Expansion (Loops)

To iterate over a list of related child records, use the hash syntax mapped to the child relationship name. The engine includes "Smart Table" detection.

If the engine finds your loop tags inside a Word Table Row (<w:tr>), it will automatically strip the tags and duplicate the entire table row for every record in the list.

{#OpportunityLineItems}
    Product: {Product2.Name} | Quantity: {Quantity} | Price: {UnitPrice}
{/OpportunityLineItems}
  1. The engine finds the boundaries of the parent table row.
  2. It duplicates that row XML for each OpportunityLineItem.
  3. Merges the individual item data into each row.

Bulk & Batch Processing

The bulk generation pipeline is designed to safely process hundreds or thousands of documents without hitting Salesforce governor limits.

Stateful Batch Execution

Generating documents requires fetching data, unzipping blobs, parsing XML, re-zipping, and making callouts for PDF renditions. Doing this in bulk triggers Heap Size (10MB) and CPU Time (10s) limits.

The Solution: The DocGenBulkController initiates a Database.Batchable class called DocGenBatch with a strict batch size of 10. This micro-batching ensures that the memory and CPU stay well below the thresholds for every chunk.

Real-Time UI Polling

The batch job incrementally updates the DocGen_Job__c record after every chunk executes (updating Success_Count__c and Error_Count__c). The LWC docGenBulkRunner polls this record every 3 seconds to animate a live progress bar for the user.

Zero-Config Background PDF Engine

Generating PDFs asynchronously (in Batches or Flows) requires a specialized loopback architecture, as the native file rendition services are not directly available to standard Apex.

The Dynamic Loopback Workflow

We leverage the internal Salesforce Connect API via a secure, authenticated REST callout back to the same org.

Zero-Config Deployment: The package dynamically generates a loopback address using Url.getOrgDomainUrl().toExternalForm(). This eliminates the need for any hardcoded Named Credentials or Remote Site Settings in subscriber orgs.

Authentication Hack: Background jobs (Queueables) drop the user session. The package uses a hidden Visualforce page (DocGenSessionProvider) to generate a full-strength Session Token at runtime, which is injected into the REST callout header.

Asynchronous Retry Mechanism ("202 Accepted")

The Connect API is asynchronous. When requesting a PDF, the API may return a 202 Accepted status if Salesforce servers are still processing the layout.

To handle this gracefully, the DocGenRenditionQueueable class incorporates a recursive retry pattern. If a 202 status is encountered, it caches the file ID and re-enqueues itself (up to 3 times) until the remote server returns a 200 OK with the PDF blob.

Invocable Flows

Integrate document generation automatically into your business logic using standard Salesforce Flows.

1. Generate Document (Native)

Use in Record-Triggered, Screen, or Autolaunched flows to generate a document for a single record.

  • Class: DocGenFlowAction
  • Inputs: Template ID, Base Record ID
  • Outputs: Content Document ID, Error Message
  • PDF Support: Yes. Automatically enqueues the rendition service at the end of the flow transaction.

2. Generate Bulk Documents

Ideal for Scheduled Flows (e.g., generating end-of-month invoices).

  • Class: DocGenBulkFlowAction
  • Inputs: Template ID, Query Condition (Optional WHERE clause, e.g., StageName='Closed Won')
  • Outputs: Job ID (ID of the DocGen_Job__c record)
  • Execution: Spawns the heavily optimized DocGenBatch job in the background to safely process all matching records.

Data Schema

DocGen_Template__c

The primary container for a document template definition.

FieldDescription
Base_Object_API__cThe Salesforce object this template merges onto.
Query_Config__cJSON configuration of the data lookup tree.
Document_Title_Format__cDynamic naming format (e.g., {Name}_Contract_{Today}).
Output_Format__cPDF, DOCX, etc.

DocGen_Job__c

Tracks the status of asynchronous bulk generation requests.

FieldDescription
Status__cQueued, Processing, Completed, Completed with Errors, Failed.
Total_Records__cTotal records targeted in the batch.
Success_Count__cSuccessful generations.
Error_Count__cFailed generations.

Apex API Reference

Developers can trigger document generation directly via Apex.

Synchronous Generation

// Generates the document and saves it to the Record ID
Id contentDocId = DocGenService.generateDocument(templateId, recordId);

// You MUST call this after generateDocument() if Output Format is PDF
// It collects all PDFs generated in the transaction and fires one async callout job
DocGenRenditionService.enqueueRenditions();

Bulk Generation via Apex

// Submits a batch job for all records matching the SOQL condition
Id jobId = DocGenBulkController.submitJob(templateId, 'StageName = \'Closed Won\'');
System.debug('Track progress on Job ID: ' + jobId);
